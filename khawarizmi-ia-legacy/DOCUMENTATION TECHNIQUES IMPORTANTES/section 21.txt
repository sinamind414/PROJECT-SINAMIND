```
CONSTAT DE DÉPART :
La plupart des apps corrigent les erreurs.
Aucune ne CLASSE les erreurs.
Or en maths, il existe exactement 4 types d'erreurs.
Chaque type a un traitement différent.

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  LES 4 TYPES D'ERREURS EN MATHS                                             │
│                                                                             │
│  TYPE 1 : ERREUR DE CONCEPT (la plus grave)                                 │
│  ─────────────────────────────────────────                                  │
│  Symptôme : "Je ne sais pas ce qu'est une dérivée."                       │
│             L'élève ne sait pas par où commencer.                          │
│  Traitement : Feynman OBLIGATOIRE avant tout exercice.                     │
│              "Explique-moi la dérivée comme à un enfant de 10 ans."       │
│  SM-2 : Reset total. Intervalle = 1 jour.                                  │
│  Urgence : HAUTE                                                            │
│                                                                             │
│  TYPE 2 : ERREUR DE MÉTHODE                                                 │
│  ─────────────────────────────                                              │
│  Symptôme : "Je sais ce qu'est une dérivée mais je l'applique              │
│              quand il ne faut pas."                                        │
│  Traitement : Drill "Quel outil ?" (sans calcul).                         │
│              10 exercices : "Quelle méthode ici ?" → pas de résolution.   │
│  SM-2 : Pas de reset. Continuer.                                           │
│  Urgence : MOYENNE                                                          │
│                                                                             │
│  TYPE 3 : ERREUR D'EXÉCUTION                                                │
│  ────────────────────────────                                               │
│  Symptôme : "Je sais la méthode mais je fais une erreur                   │
│              de calcul à l'étape 3."                                       │
│  Traitement : Drill "Détection de piège" ciblé.                           │
│              Focus : vérification systématique de chaque étape.           │
│  SM-2 : Pas de reset.                                                      │
│  Urgence : BASSE                                                            │
│                                                                             │
│  TYPE 4 : ERREUR DE PANIQUE                                                 │
│  ────────────────────────────                                               │
│  Symptôme : "Je sais faire à la maison. En examen, je bloque."            │
│  Traitement : Mode Examen Simulé. 45 min. Chrono. Aucune aide.            │
│  SM-2 : Pas de reset.                                                      │
│  Urgence : MOYENNE                                                          │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  LE PROTOCOLE ERROR AUTOPSY :                                               │
│                                                                             │
│  Après CHAQUE erreur, AVANT la correction :                                │
│                                                                             │
│  Q1 : "Est-ce que tu savais QUE faire ?"      (Oui / Non)                 │
│  Q2 : "Est-ce que tu savais COMMENT faire ?"  (Oui / Non)                 │
│  Q3 : "Est-ce que c'est une erreur de CALCUL ?" (Oui / Non)              │
│                                                                             │
│  Q1 = Non                      → TYPE 1 → Feynman ce soir                │
│  Q1 = Oui / Q2 = Non          → TYPE 2 → Drill méthode                   │
│  Q1 = Oui / Q2 = Oui / Q3=Oui → TYPE 3 → Drill calcul ciblé             │
│  Q1 = Oui / Q2 = Oui / Q3=Non → TYPE 4 → Mode examen                    │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  PONDÉRATION DE L'ÉVALUATION MATHS :                                        │
│                                                                             │
│  70% de la note = RAISONNEMENT (méthode et étapes)                        │
│  30% de la note = RÉSULTAT FINAL                                           │
│                                                                             │
│  Un bon raisonnement avec erreur de calcul = 6/10                          │
│  Un résultat correct sans méthode = 3/10                                   │
│                                                                             │
│  PRINCIPE : On récompense le RAISONNEMENT, pas la chance.                  │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  RÉPARTITION DES SESSIONS (Maths) :                                         │
│  → 70% : Drill étape par étape (exercices annales réels)                  │
│  → 20% : Chat Socratique (quand l'élève bloque)                           │

│                                                                             │

B2. PHYSIQUE-CHIMIE — LA FORMULE "SENS AVANT CALCUL"
──────────────────────────────────────────────────────────────────

```
CONSTAT DE DÉPART :
99% des élèves algériens font la physique à l'ENVERS.
Ce qu'ils font  : Énoncé → Formule → Calcul → Résultat
Ce qu'il faut   : Énoncé → Sens physique → Estimation → Formule → Vérification

La différence entre 12/20 et 17/20 en physique
n'est pas le calcul. C'est l'INTUITION PHYSIQUE.

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  LE PROTOCOLE "SENS AVANT CALCUL" — 5 étapes obligatoires                  │
│                                                                             │
│  ÉTAPE 1 : DESSIN (30 secondes)                                             │
│  → Schématiser la situation physique                                       │
│  → Identifier les grandeurs sur le schéma                                 │
│  → Zéro chiffre à cette étape                                              │
│  → L'élève décrit son schéma en 1-2 phrases                               │
│                                                                             │
│  ÉTAPE 2 : SENS PHYSIQUE (30 secondes)                                      │
│  → "Que se passe-t-il PHYSIQUEMENT ici ?"                                 │
│  → Pas de formules. Des mots simples.                                     │
│  → "Un condensateur se décharge dans un circuit                           │
│     avec bobine et résistance. La tension va diminuer."                   │
│  → IA évalue : a-t-il décrit le phénomène correctement ?                 │
│  → Score /3 : 3=parfait, 2=partiel, 1=confus, 0=formule récitée           │
│                                                                             │
│  ÉTAPE 3 : ESTIMATION (30 secondes)                                         │
│  → Avant tout calcul : ordre de grandeur du résultat ?                   │
│  → □ Milliampères  □ Ampères  □ Centaines d'ampères                       │
│  → Unité probable : ________                                              │
│  → Il n'y a pas de mauvaise estimation.                                   │
│    L'objectif = réfléchir avant de calculer.                              │
│  → L'estimation est NOTÉE et COMPARÉE après calcul.                       │
│                                                                             │
│  ÉTAPE 4 : CALCUL (2-5 minutes)                                             │
│  → Appliquer la formule avec les unités                                   │
│  → Montrer chaque étape                                                    │
│                                                                             │
│  ÉTAPE 5 : VÉRIFICATION DU SENS (30 secondes)                              │
│  → Le résultat est-il proche de l'estimation ?                            │
│  → Physiquement cohérent ?                                                 │
│    (500 km/s pour une voiture = IMPOSSIBLE → reprendre)                   │
│  → Les unités sont-elles correctes ?                                      │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  VÉRIFICATION DES UNITÉS — Automatique et obligatoire                       │
│                                                                             │
│  → Réponse sans unité = score max 5/10                                     │
│  → Unité fausse = explication ciblée de l'IA                              │
│  → Ordre de grandeur absurde = IA signale AVANT que l'élève passe         │
│                                                                             │
│  EXEMPLES DE PIÈGES D'UNITÉS (pré-enregistrés) :                          │
│  → km/h → m/s : diviser par 3.6 (erreur #1 en cinématique)              │
│  → kΩ → Ω : multiplier par 1000 (erreur #1 en électricité)               │
│  → min → s : multiplier par 60 (erreur fréquente en chimie)               │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  PONDÉRATION DE L'ÉVALUATION PHYSIQUE :                                     │
│                                                                             │
│  30% : Choix de la bonne formule/loi                                       │
│  25% : Justification du choix (POURQUOI cette loi)                        │
│  25% : Calcul correct avec étapes                                          │
│  10% : Unités présentes et correctes                                       │
│  10% : Cohérence physique du résultat                                      │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  RÉPARTITION DES SESSIONS (Physique) :                                      │
│  → 50% : Drill formules appliquées (annales réelles)                      │
│  → 25% : Feynman (expliquer les phénomènes physiques)                    │
│  → 15% : Chat Socratique (guidage vers la bonne loi)                     │
│  → 10% : Cued Recall SM-2 (formules clés à mémoriser)                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

CONSTAT DE DÉPART :
Les élèves mémorisent les processus NORMAUX.
Le BAC pose des questions sur les PANNES.

"Décris la mitose" → Tout le monde peut réciter.
"Si la séparase est défectueuse, que se passe-t-il ?"
→ Seul celui qui COMPREND la logique peut répondre.

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  LE PROTOCOLE "PROCESSUS → PANNE" — 3 niveaux                              │
│                                                                             │
│  NIVEAU 1 : MAÎTRISER LE PROCESSUS NORMAL                                  │
│  → Quelles sont les étapes dans l'ordre ?                                 │
│  → Quel est le rôle de chaque acteur (molécule, organe, enzyme) ?        │
│  → Free Recall : "Décris les 4 phases de la mitose dans l'ordre."         │
│  → SM-2 : révision espacée jusqu'à niveau 4                               │
│                                                                             │
│  NIVEAU 2 : SIMULER LA PANNE (le cœur de la méthode)                      │
│  → "Et si l'étape 2 ne fonctionnait pas ?"                               │
│  → "Et si cette enzyme était absente ?"                                   │
│  → "Et si ce récepteur était bloqué ?"                                    │
│  → L'élève doit DÉDUIRE les conséquences en cascade                       │
│  → L'IA évalue : sont-elles dans le bon ordre ? Sont-elles complètes ?   │
│                                                                             │
│  NIVEAU 3 : RELIER À LA PATHOLOGIE RÉELLE                                  │
│  → Quelle maladie réelle correspond à cette panne ?                       │
│  → La pathologie = ancrage mémoriel puissant                              │
│  → "Séparase défectueuse → aneuploïdie → cancer"                         │
│  → "LB absents → pas d'anticorps → agammaglobulinémie"                   │

class PiloteAutomatique {

    static async getExerciceDuJour(userId, db) {

        // Priorité 1 : Chapitres DUS selon SM-2 (révision espacée)
        const dus = await db.query(`
            SELECT matiere, chapitre, niveau, score_actuel,
                   prochaine_revision
            FROM mastery_levels
            WHERE user_id = $1
              AND prochaine_revision <= NOW()
              AND niveau > 0
            ORDER BY prochaine_revision ASC
            LIMIT 1
        `, [userId]);

        // Priorité 2 : Chapitre le plus faible (si aucun SM-2 dû)
        const faible = await db.query(`
            SELECT matiere, chapitre, niveau, score_actuel
            FROM mastery_levels
            WHERE user_id = $1
              AND niveau IN (0, 1, 2)
            ORDER BY score_actuel ASC
            LIMIT 1
        `, [userId]);

        const cible = dus.rows[0] || faible.rows[0];

        if (!cible) {
            // L'élève est excellent partout (rare)
            return { message: "Tu as tout révisé ! Mode Examen Simulé activé." };
        }

        // Sélectionner un exercice d'annale sur ce chapitre
        const exercice = await db.query(`
            SELECT * FROM annales
            WHERE chapitre_id = $1
              AND id NOT IN (
                  SELECT question_id FROM exercise_results
                  WHERE user_id = $2
                    AND created_at > NOW() - INTERVAL '7 days'
              )
            ORDER BY RANDOM()
            LIMIT 1
        `, [cible.chapitre, userId]);

        return {
            message: `📅 Aujourd'hui : ${cible.chapitre}`,
            raison:  cible === dus.rows[0]
                     ? "SM-2 : C'est le moment de revoir ce chapitre."
                     : "Tu es faible ici. On travaille ça maintenant.",
            exercice: exercice.rows[0],
            niveau_actuel: cible.niveau,
        };
    }
}

// simulateur-moyenne.js

class SimulateurMoyenneBAC {

    static COEFFICIENTS = {
        sciences_experimentales: {
            sciences_naturelles:  6,
            physique_chimie:      5,
            mathematiques:        5,
            arabe:                3,
            philosophie:          3,
            histoire_geo:         2,
            francais:             2,
            anglais:              2,
            eps:                  1,
        },
        mathematiques: {
            mathematiques:        7,
            physique_chimie:      6,
            sciences_naturelles:  5,
            arabe:                3,
            histoire_geo:         2,
            anglais:              2,
            philosophie:          2,
            francais:             1,
            eps:                  1,
        },
    };

    static calculerMoyenne(notes, filiere) {
        const coeffs = this.COEFFICIENTS[filiere];
        let total = 0;
        let totalCoeffs = 0;

        for (const [matiere, coeff] of Object.entries(coeffs)) {
            if (notes[matiere] !== undefined) {
                total       += notes[matiere] * coeff;
                totalCoeffs += coeff;
            }
        }

        return parseFloat((total / totalCoeffs).toFixed(2));
    }

    static simulerProgression(notes, filiere, matiereAmeliorer, gainPoints) {

        const notesActuelles = { ...notes };
        const notesAmeliorees = {
            ...notes,
            [matiereAmeliorer]: Math.min(20,
                (notes[matiereAmeliorer] || 0) + gainPoints
            ),
        };

        const moyActuelle  = this.calculerMoyenne(notesActuelles,  filiere);
        const moyAmelioree = this.calculerMoyenne(notesAmeliorees, filiere);

        return {
            moyenneActuelle:   moyActuelle,
            moyenneAmelioree:  moyAmelioree,
            gainMoyenne:       parseFloat((moyAmelioree - moyActuelle).toFixed(2)),
            statut_actuel:     moyActuelle >= 10 ? 'ADMIS' : 'REFUSÉ',
            statut_ameliore:   moyAmelioree >= 10 ? 'ADMIS' : 'REFUSÉ',
            impact:            moyActuelle < 10 && moyAmelioree >= 10
                               ? 'DÉCISIF : tu passes !'
                               : `+${(moyAmelioree - moyActuelle).toFixed(2)} points de moyenne`,
        };
    }

    // Identifier les chapitres prioritaires pour atteindre l'objectif
    static async recommanderChapitres(userId, matiereAmeliorer, db) {

        const chapitresFaibles = await db.query(`
            SELECT chapitre, score_actuel, niveau
            FROM mastery_levels
            WHERE user_id = $1
              AND matiere = $2
              AND niveau < 3
            ORDER BY score_actuel ASC
            LIMIT 3
        `, [userId, matiereAmeliorer]);

        return {
            message: `Pour gagner des points en ${matiereAmeliorer}, 
                      concentre-toi sur ces chapitres :`,
            chapitres: chapitresFaibles.rows.map(c => ({
                nom:          c.chapitre,
                score_actuel: Math.round(c.score_actuel * 100) + '%',
                priorite:     c.niveau === 0 ? '🔴 URGENT'
                            : c.niveau === 1 ? '🟠 IMPORTANT'
                            : '🟡 À CONSOLIDER',
            })),
        };
    }
}
