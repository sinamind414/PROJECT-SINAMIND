SECTION 4 : PIPELINE RAG (CERVEAU DE KHAWARIZMI)
================================================================================

SOURCES → EXTRACTION → CHUNKING → EMBEDDING → STOCKAGE → RETRIEVAL

1. SOURCES :
   ├── Manuels 3AS officiels (toutes filières, toutes matières)
   ├── Annales BAC ONEC 2010-2025
   ├── Corrections types ONEC
   └── Volume : 500+ documents PDF

2. EXTRACTION :
   ├── PDF texte natif → PyMuPDF (fitz)
   ├── PDF scanné → Tesseract OCR (fra + ara)
   ├── Formules math → extraction regex, conservation LaTeX
   └── Nettoyage : artefacts ONEC, watermarks, pagination

3. CHUNKING SÉMANTIQUE :
   ├── PAS fixe (500 chars) → casse les idées
   ├── Par structure du document :
   │   ├── Définitions    → 150 tokens, chunk isolé
   │   ├── Formules       → 100 tokens, chunk isolé
   │   ├── Cours/Section  → 400 tokens, avec overlap 50
   │   ├── Exercices      → 600 tokens, complet avec données
   │   ├── Corrections    → 800 tokens, step-by-step complet
   │   └── Annales BAC    → 1000 tokens, par Partie/Exercice
   ├── Métadonnées par chunk :
   │   matière, filière, chapitre, sous-chapitre, type,
   │   année_bac, difficulté, langue
   └── Volume estimé : ~50 000 chunks

4. EMBEDDING :
   ├── Modèle : text-embedding-3-small (1536 dimensions)
   ├── Texte enrichi : métadonnées + contenu dans l'embedding
   ├── Coût TOTAL : ~$0.40 pour indexer tout le programme
   └── Batch async : 100 chunks/batch

5. STOCKAGE :
   ├── pgvector dans PostgreSQL (pas Pinecone = trop cher)
   ├── Index IVFFlat (lists=100) cosine similarity
   ├── Filtres SQL : matière, filière, type_chunk, année
   ├── Fonction : search_curriculum(embedding, filtres, top_k)
   └── Seuil pertinence : similarité > 0.75

6. RETRIEVAL TEMPS RÉEL :
   ├── Question élève → embedding question
   ├── Recherche vectorielle + filtres → Top 5 chunks
   ├── Classification complexité → choix modèle (Nano/Mini/Flash)
   ├── Prompt = System DZ + contexte chunks + question
   ├── Réponse ancrée programme algérien
   ├── Score confiance (< 0.75 = avertissement)
   └── Logging interaction pour amélioration continue

SECTION 5 : 7 MÉTHODES SCIENTIFIQUES IMPLÉMENTÉES
================================================================================

91% des étudiants utilisent la relecture passive = INEFFICACE
KHAWARIZMI force les méthodes PROUVÉES par la recherche

MÉTHODE 1 — ACTIVE RECALL (Rappel Actif)
Stat    : +80% rétention vs relecture (Roediger & Karpicke 2006)
          +73% scores (méta-analyse 217 études, Toronto 2024)
Feature : Quiz en 3 phases (QCM → ouvert → libre)
          L'élève PRODUIT la réponse, ne la RECONNAÎT pas

MÉTHODE 2 — SPACED REPETITION (Répétition Espacée)
Stat    : Empêche 90% de l'oubli (Ebbinghaus)
          82% études confirment supériorité (Cepeda, 317 études)
Feature : Algorithme SM-2 adapté, notifications quotidiennes
          Intervalles : 1j → 3j → 7j → 18j → ...

MÉTHODE 3 — INTERLEAVING (Entrelacement)
Stat    : 6x plus efficace vs blocked practice
          +21 points examens (Rohrer & Taylor 2007)
Feature : Sessions mixtes, chapitres mélangés par l'IA
          Max 2 exercices consécutifs du même chapitre

MÉTHODE 4 — FEYNMAN TECHNIQUE
Stat    : Gains d'apprentissage significatifs (p<0.05)
Feature : "Explique ce concept" → IA évalue vs cours officiel
          Score /10 + lacunes identifiées + conseils

MÉTHODE 5 — DUAL CODING (Double Codage)
Stat    : 65% apprennent mieux visuellement
          Dual coding : ~75% rétention vs 40% texte seul
Feature : Chaque explication = texte + schéma Mermaid.js auto

MÉTHODE 6 — ELABORATIVE INTERROGATION
Stat    : +40% compréhension profonde
Feature : Questions "POURQUOI" auto-générées après chaque formule
          "Que se passe-t-il si X double ?"

MÉTHODE 7 — DESIRABLE DIFFICULTY
Stat    : Zone optimale 50-70% réussite = apprentissage max
Feature : Difficulté auto-ajustée selon score de l'élève
          Trop facile (>80%) → augmente | Trop dur (<40%) → réduit
