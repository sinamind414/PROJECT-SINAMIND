L'objectif est d'analyser les réponses de l'élève aux quiz de Rappel Actif pour diagnostiquer avec précision ses lacunes.

* **Mécanique technique :** Lorsqu'un élève tape ou dicte une réponse, ton backend transforme cette réponse en un vecteur mathématique (via un modèle d'*Embeddings*). L'IA compare ensuite ce vecteur au vecteur de la "bonne réponse" issue de ta base de données officielle.
* **La mesure :** Nous utilisons la Similarité Cosinus. Si la similarité entre la réponse de l'élève et la base certifiée est inférieure à un seuil prédéfini (ex: 70%), l'IA identifie instantanément une "anomalie de compréhension" sur ce sous-chapitre précis.

### 2. Le Tri et l'Orientation (Le *Routing* IA Bilingue)

L'IA agit comme un "Coach" Premium qui évalue le niveau et oriente vers le parcours approprié.

### 3. La Prédiction de la Note (Le *Machine Learning* Prédictif)

En analysant les métriques de l'élève (temps de réponse, taux d'erreur, régularité), le système identifie les lycéens à haut risque d'échec.

* **Mécanique technique :** Cette brique ne dépend pas de l'IA générative (LLM), mais de modèles d'apprentissage automatique classiques (comme *XGBoost* ou *Random Forest*).
* **Le traitement :** Ton application doit enregistrer méticuleusement chaque interaction dans une base de données relationnelle (PostgreSQL). Le modèle prédictif calcule une fonction de probabilité de réussite au BAC $P(BAC)$ basée sur des variables temporelles et comportementales :

$$P(BAC) = f(E_{rate}, T_{resp}, R_{freq})$$



*(Où $E$ est le taux d'erreur, $T$ le temps de réponse moyen, et $R$ la fréquence de révision).*

### 4. L'Allègement Cognitif (Génération JSON & Structuration)

L'IA extrait les informations du texte officiel pour générer automatiquement des flashcards (Répétition Espacée) et des cartes mentales.

* 
**Mécanique technique :** C'est ici que l'architecture RAG brille. Le serveur récupère un bloc de leçon vérifié et l'envoie à l'IA avec un prompt d'extraction strict.


* 
**Les formats de sortie :** * Pour les cartes mentales, l'IA génère des plans ultra-structurés  sous forme de code `Markdown` ou `Mermaid.js` que ton application affiche visuellement.


* Pour les flashcards, l'IA est forcée de répondre sous format `JSON` structuré (ex: `{"question": "...", "reponse": "..."}`) pour peupler instantanément la base de données de l'élève sans erreur.





### 5. La Gestion de la Mémoire (L'Algorithme Spaced Repetition)

L'IA utilise la Répétition Espacée pour anticiper le pic de l'oubli naturel du cerveau de l'élève. Elle programme la réapparition d'une question difficile au moment exact où l'élève est sur le point de l'oublier.

* **Mécanique technique :** Ton backend doit implémenter un algorithme de mémorisation reconnu, comme le *SuperMemo-2 (SM-2)*.
* **L'équation :** Chaque flashcard possède un facteur de facilité (EF). Selon la qualité de la réponse de l'élève (notée de 0 à 5), le prochain intervalle de révision $I(n)$ en jours est recalculé mathématiquement :

$$I(n) = I(n-1) \times EF$$



Si l'élève échoue, l'intervalle retombe à zéro. S'il réussit, la carte est repoussée de plus en plus loin dans le temps.

# routing.py — Ajouter à notre moteur adaptatif

def router_par_niveau(niveau: int, score_actuel: float) -> str:
    """
    Décide quelle méthode pédagogique activer.
    Basé sur le niveau SM-2 (0-4) et le score actuel (0-1).
    """
    if niveau <= 1 or score_actuel < 0.40:
        # Détresse → Feynman pour comprendre le concept de base
        return 'FEYNMAN'

    elif niveau == 2 or score_actuel < 0.70:
        # En progression → Rappel actif sur annales récentes
        return 'RAPPEL_ACTIF'

    elif niveau == 3 or score_actuel < 0.90:
        # Maîtrisé → Annales difficiles + Error Autopsy
        return 'ANNALES_COMPLEXES'

    else:
        # Expert → Annales BAC 2024 les plus récentes, mode examen
        return 'MODE_EXAMEN'
