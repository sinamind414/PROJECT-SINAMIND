## LA SOLUTION DE L'ÉTAT DE L'ART 2026 — Le pipeline en 2 étapes

La recherche la plus récente (UC Irvine, Mars 2026) a convergé vers une architecture que je considère comme la meilleure approche pour Khawarizmi :

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  LE PIPELINE 2 ÉTAPES (validé par la recherche)                            │
│  ═══════════════════════════════════════════════                             │
│                                                                             │
│  ÉTAPE 1 : TRANSCRIRE (séparer la lecture de l'évaluation)                │
│  ──────────────────────────────────────────────────────────                │
│                                                                             │
│  → L'IA reçoit UNIQUEMENT l'image                                         │
│  → Mission : "Transcris ce que l'élève a écrit. Rien d'autre."           │
│  → Output : texte + LaTeX des formules                                    │
│  → Si des parties sont illisibles → le SIGNALER (pas deviner)            │
│  → PAS d'évaluation. PAS de score. Juste la transcription.              │
│                                                                             │
│  ÉTAPE 2 : ÉVALUER (à partir du texte transcrit)                          │
│  ──────────────────────────────────────────────────                         │
│                                                                             │
│  → L'IA reçoit : le TEXTE transcrit (pas l'image)                        │
│    + la solution officielle + le barème + le profil de l'élève           │
│  → Mission : "Compare la réponse transcrite à la solution officielle.    │
│    Note chaque étape. Identifie les erreurs."                            │
│  → Output : score + feedback détaillé par étape                          │
│                                                                             │
│  POURQUOI C'EST MEILLEUR QUE LE TOUT-EN-UN :                              │
│                                                                             │
│  → L'IA de l'étape 1 se concentre à 100% sur la LECTURE                  │
│    (pas distraite par l'évaluation)                                      │
│  → L'IA de l'étape 2 travaille sur du TEXTE PROPRE                      │
│    (pas perturbée par l'écriture)                                        │
│  → Si l'étape 1 signale une partie illisible                             │
│    → L'étape 2 peut le prendre en compte                                │
│    (au lieu de deviner silencieusement)                                  │
│  → L'élève peut VÉRIFIER la transcription                                │
│    avant que l'évaluation ne commence                                    │
│  → Transparence totale                                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## L'IMPLÉMENTATION KHAWARIZMI — Pipeline corrigé

```javascript
// paper-correction-pipeline.js — Pipeline en 2 étapes (état de l'art 2026)

class PaperCorrectionPipeline {

    // ═══ ÉTAPE 1 : TRANSCRIRE ═══════════════════════════════

    static async transcrire(imageBase64) {

        const promptTranscription = `
Tu es un expert en lecture d'écriture manuscrite mathématique.

MISSION : Transcris EXACTEMENT ce que l'élève a écrit sur cette photo.
         Rien d'autre. Pas de correction. Pas d'évaluation.

RÈGLES :
→ Utilise le format LaTeX pour les formules
→ Garde EXACTEMENT ce que l'élève a écrit (même si c'est faux)
→ Si une partie est illisible → écris [ILLISIBLE: description]
→ Si un symbole est ambigu → écris [AMBIGU: "3 ou 8?"]
→ Note les ratures : l'élève a barré X et écrit Y → garde Y
→ Numérote chaque ligne de la copie
→ Signale les schémas/dessins : [SCHÉMA: description]

RÉPONDS EN JSON :
{
    "transcription_lignes": [
        {"ligne": 1, "contenu": "<texte LaTeX>", "lisibilite": "bon|moyen|illisible"},
        {"ligne": 2, "contenu": "<texte LaTeX>", "lisibilite": "bon"}
    ],
    "parties_illisibles": [
        {"ligne": 3, "description": "Un chiffre entre le 3 et le 8"}
    ],
    "schemas_detectes": [
        {"position": "en bas à droite", "description": "Graphe de fonction"}
    ],
    "qualite_globale": "bon|moyen|mauvais",
    "confiance_transcription": 0.92
}

NE CORRIGE RIEN. NE JUGE RIEN. TRANSCRIS SEULEMENT.
        `;

        const response = await callIA('gpt-4o-mini', {
            messages: [{
                role: 'user',
                content: [
                    { type: 'text', text: promptTranscription },
                    { type: 'image_url', image_url: {
                        url: `data:image/jpeg;base64,${imageBase64}`,
                        detail: 'high'
                    }},
                ],
            }],
            temperature: 0.1,
            response_format: { type: 'json_object' },
        });

        return JSON.parse(response);
    }

    // ═══ ÉTAPE 1.5 : VÉRIFICATION PAR L'ÉLÈVE ═══════════════

    static genererEcranVerification(transcription) {
        // SI confiance < 0.85 OU parties illisibles détectées
        // → Montrer la transcription à l'élève AVANT d'évaluer

        if (transcription.confiance_transcription < 0.85
            || transcription.parties_illisibles.length > 0) {

            return {
                action: 'VERIFICATION_REQUISE',
                message: `📝 J'ai lu ta copie. Vérifie que j'ai bien compris :`,
                transcription_affichee: transcription.transcription_lignes,
                parties_a_clarifier: transcription.parties_illisibles,
                boutons: [
                    "✅ C'est correct, évalue ma réponse",
                    "❌ Ce n'est pas ce que j'ai écrit → reprendre la photo",
                    "✏️ Corriger un mot/chiffre spécifique"
                ],
            };
        }

        // Si confiance élevée et pas de partie illisible
        // → Passer directement à l'évaluation
        return { action: 'EVALUER_DIRECTEMENT' };
    }

    // ═══ ÉTAPE 2 : ÉVALUER (sur le texte, PAS sur l'image) ══

    static async evaluer(
        transcription,    // Le texte transcrit (étape 1)
        exercice,         // L'exercice original
        solutionOfficielle, // La correction officielle
        profilEleve,      // ADN erreur + niveau
        matiere           // Pour adapter le prompt
    ) {
        const promptsParMatiere = {

            mathematiques: `
Tu es KHAWARIZMI, correcteur expert du BAC algérien en Mathématiques.

EXERCICE : ${exercice.question_text}

SOLUTION OFFICIELLE :
${JSON.stringify(solutionOfficielle)}

CE QUE L'ÉLÈVE A ÉCRIT (transcription de sa copie manuscrite) :
${transcription.transcription_lignes.map(l =>
    `Ligne ${l.ligne}: ${l.contenu}`
).join('\n')}

PARTIES ILLISIBLES SIGNALÉES :
${transcription.parties_illisibles.map(p =>
    `Ligne ${p.ligne}: ${p.description}`
).join('\n') || 'Aucune'}

BARÈME : ${exercice.points} points total

PROFIL ÉLÈVE :
- Niveau sur ce concept : ${profilEleve.niveau}/4
- Erreurs fréquentes : ${profilEleve.erreurs_frequentes}

ÉVALUE CHAQUE ÉTAPE DE LA SOLUTION ATTENDUE :

Pour chaque étape :
1. L'élève l'a-t-il faite ? (Oui/Non/Partiellement)
2. Est-elle correcte ? (Oui/Non)
3. Si incorrecte : cite EXACTEMENT la ligne de l'élève qui contient l'erreur
4. Type d'erreur : TYPE_1 (concept) / TYPE_2 (méthode) / TYPE_3 (calcul) / TYPE_4 (omission)
5. Points accordés pour cette étape

BARÈME DE NOTATION :
→ 70% raisonnement (méthode et étapes)
→ 30% résultat final
→ Un bon raisonnement avec erreur de calcul = 60-70% des points
→ Un résultat correct sans méthode visible = 30% des points max

SI PARTIE ILLISIBLE :
→ NE PAS deviner. Dire "Partie illisible — non évaluée."
→ NE PAS pénaliser l'élève pour une partie illisible.
→ Suggérer : "Reprends la photo de cette partie."

RÉPONDS EN JSON :
{
    "score_total": <0 à ${exercice.points}>,
    "score_pourcentage": <0 à 100>,
    "evaluation_etapes": [
        {
            "etape_attendue": "<nom>",
            "faite": true/false,
            "correcte": true/false,
            "ligne_eleve": <numéro de ligne dans la transcription>,
            "citation_eleve": "<ce que l'élève a écrit à cette étape>",
            "erreur": "<description précise si incorrecte>",
            "type_erreur": "TYPE_1/2/3/4/AUCUNE",
            "points_accordes": <nombre>,
            "points_max": <nombre>
        }
    ],
    "feedback_positif": "<commencer par ce qui est BIEN>",
    "feedback_erreurs": "<erreurs spécifiques avec référence aux lignes>",
    "conseil_progres": "<1 conseil actionnable>",
    "type_erreur_dominant": "TYPE_1/2/3/4/AUCUNE"
}

RÈGLES ABSOLUES :
→ Cite TOUJOURS la ligne exacte de l'élève (pas de feedback vague)
→ Commence TOUJOURS par ce qui est bien (bienveillance)
→ Accepte les variantes mathématiques équivalentes
→ Ne DEVINE JAMAIS une partie illisible
            `,

            physique: `
[Même structure mais avec vérification des unités + cohérence physique]
→ Ajouter : "Vérifie que les unités sont présentes et cohérentes"
→ Ajouter : "Le résultat est-il physiquement réaliste ?"
            `,

            sciences_naturelles: `
[Même structure mais avec vocabulaire strict + processus]
→ Ajouter : "Le terme scientifique exact est-il utilisé ?"
→ Ajouter : "Les étapes du processus sont-elles dans le bon ordre ?"
            `,
        };

        const response = await callIA('deepseek-chat', {
            messages: [{
                role: 'user',
                content: promptsParMatiere[matiere],
            }],
            temperature: 0.1,
            response_format: { type: 'json_object' },
        });

        return JSON.parse(response);
    }

    // ═══ PIPELINE COMPLET ════════════════════════════════════

    static async corriger(imageBase64, exercice, solution, profil, matiere) {

        // ÉTAPE 0 : Vérifier la qualité de la photo
        const qualite = await PhotoQualityCheck.verifier(imageBase64);
        if (!qualite.qualite_ok) {
            return {
                action: 'REPRENDRE_PHOTO',
                message: qualite.diagnostics[0].message,
            };
        }

        // ÉTAPE 1 : Transcrire
        const transcription = await this.transcrire(imageBase64);

        // ÉTAPE 1.5 : Vérification si nécessaire
        const verif = this.genererEcranVerification(transcription);
        if (verif.action === 'VERIFICATION_REQUISE') {
            return verif; // L'app demande confirmation à l'élève
        }

        // ÉTAPE 2 : Évaluer (sur le texte, PAS l'image)
        const evaluation = await this.evaluer(
            transcription, exercice, solution, profil, matiere
        );

        // ÉTAPE 3 : Mettre à jour la base de données
        // (SM-2, ADN erreur, XP — comme avant, pas de changement)

        return {
            action: 'RESULTAT',
            transcription,
            evaluation,
        };
    }
}
```

## LA SOLUTION — Le système anti-triche bienveillant

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  PRINCIPE : L'élève peut corriger la TRANSCRIPTION (si l'IA a mal lu).    │
│  L'élève ne peut PAS corriger sa RÉPONSE (triche).                         │
│                                                                             │
│  La différence :                                                            │
│  → "L'IA a lu 3 au lieu de 8" = erreur de transcription → CORRIGER OK    │
│  → "J'ai écrit 7 mais je voulais écrire 5" = triche → BLOQUER            │
│                                                                             │
│  COMMENT DISTINGUER LES DEUX ?                                              │
│                                                                             │

│  → Le mensonge ne marche pas car SM-2 est honnête                        │

│  → L'impulsion de tricher passe dans les 5 premières secondes            │

│  → La majorité clique "✅ C'est ma réponse"                              │

## L'IMPLÉMENTATION CODE

```javascript
// photo-confirmation.js — Système de confirmation avec anti-triche

class PhotoConfirmation {

    static MAX_REPRISES = 2;

    static async processPhoto(userId, exerciceId, imageBase64, tentative, db) {

        // ═══ GARDER TOUTES LES PHOTOS ═══════════════════════
        // La 1ère photo est TOUJOURS conservée (jamais écrasée)

        await db.query(`
            INSERT INTO photo_history
                (user_id, exercice_id, tentative, image_hash,
                 created_at)
            VALUES ($1, $2, $3, $4, NOW())
        `, [userId, exerciceId, tentative, hashImage(imageBase64)]);

        // ═══ ÉTAPE 1 : TRANSCRIRE ═══════════════════════════
        const transcription = await PaperCorrectionPipeline.transcrire(imageBase64);

        // ═══ STOCKER LA TRANSCRIPTION PAR TENTATIVE ══════════
        await db.query(`
            INSERT INTO transcription_history
                (user_id, exercice_id, tentative,
                 transcription, confiance, created_at)
            VALUES ($1, $2, $3, $4, $5, NOW())
        `, [
            userId,
            exerciceId,
            tentative,
            JSON.stringify(transcription),
            transcription.confiance_transcription,
        ]);

        // ═══ RETOURNER L'ÉCRAN DE CONFIRMATION ════════════════
        return {
            action:         'CONFIRMATION_REQUISE',
            transcription:  transcription,
            tentative:      tentative,
            timer_secondes: 15,

            boutons: tentative < this.MAX_REPRISES
                ? {
                    confirmer: {
                        texte:  "✅ Oui c'est ma réponse → Corriger",
                        actif:  true,
                    },
                    reprendre: {
                        texte:  "📸 L'IA a mal lu → Reprendre",
                        actif:  false,  // Grisé pendant 15 secondes
                        delai:  15,     // S'active après 15s
                        note:   "Reprends seulement si l'IA a mal lu ton écriture.",
                    },
                }
                : {
                    // 3ème tentative → pas de bouton "reprendre"
                    confirmer: {
                        texte:  "✅ Confirme ta réponse",
                        actif:  true,
                    },
                    message:  "Tu as repris la photo 2 fois. " +
                              "C'est OK de se tromper ! " +
                              "Confirme et laisse-moi t'aider à comprendre.",
                },
        };
    }

    // ═══ QUAND L'ÉLÈVE CONFIRME ══════════════════════════════

    static async onConfirmation(userId, exerciceId, tentative, db) {

        // Récupérer la transcription confirmée
        const trans = await db.query(`
            SELECT transcription FROM transcription_history
            WHERE user_id = $1 AND exercice_id = $2
              AND tentative = $3
        `, [userId, exerciceId, tentative]);

        // ═══ DÉTECTION DE TRICHE SILENCIEUSE ═══════════════
        // Comparer avec la 1ère tentative (si c'est pas la 1ère)

        let scoreInterne = null;

        if (tentative > 1) {
            const premiereTrans = await db.query(`
                SELECT transcription FROM transcription_history
                WHERE user_id = $1 AND exercice_id = $2
                  AND tentative = 1
            `, [userId, exerciceId]);

            const diff = this.comparerTranscriptions(
                JSON.parse(premiereTrans.rows[0].transcription),
                JSON.parse(trans.rows[0].transcription)
            );

            if (diff.correction_detectee) {
                // L'élève a changé sa réponse (pas juste la qualité photo)
                // → NE RIEN DIRE (bienveillant)
                // → MAIS utiliser le score de la 1ère réponse pour SM-2

                scoreInterne = 'UTILISER_PREMIERE_REPONSE';

                await db.query(`
                    INSERT INTO triche_log
                        (user_id, exercice_id, type,
                         transcription_v1, transcription_v2, created_at)
                    VALUES ($1, $2, 'correction_retroactive', $3, $4, NOW())
                `, [
                    userId, exerciceId,
                    premiereTrans.rows[0].transcription,
                    trans.rows[0].transcription,
                ]);
            }
        }

        // ═══ PASSER À L'ÉVALUATION ═══════════════════════════
        const evaluation = await PaperCorrectionPipeline.evaluer(
            JSON.parse(trans.rows[0].transcription),
            // ... exercice, solution, profil, matière
        );

        // ═══ SM-2 : UTILISER LE VRAI SCORE ═══════════════════
        if (scoreInterne === 'UTILISER_PREMIERE_REPONSE') {
            // Évaluer aussi la 1ère réponse (en arrière-plan)
            const premiereTrans = await db.query(`
                SELECT transcription FROM transcription_history
                WHERE user_id = $1 AND exercice_id = $2
                  AND tentative = 1
            `, [userId, exerciceId]);

            const evalPremiere = await PaperCorrectionPipeline.evaluer(
                JSON.parse(premiereTrans.rows[0].transcription),
                // ... même exercice, même solution
            );

            // SM-2 utilise le score de la PREMIÈRE réponse
            await SM2.updateAfterExercise(
                userId,
                exerciceId,
                evalPremiere.score_pourcentage,  // ← le VRAI score
                db
            );

            // MAIS afficher le score de la DERNIÈRE réponse
            return {
                score_affiche:  evaluation.score_pourcentage,  // ← ce que l'élève voit
                score_interne:  evalPremiere.score_pourcentage, // ← ce que SM-2 utilise
                feedback:       evaluation.feedback_positif,
                // L'élève est content. SM-2 est honnête.
            };
        }

        // Cas normal (pas de triche détectée)
        await SM2.updateAfterExercise(
            userId, exerciceId,
            evaluation.score_pourcentage,
            db
        );

        return evaluation;
    }

    // ═══ COMPARAISON DES TRANSCRIPTIONS ═════════════════════

    static comparerTranscriptions(trans1, trans2) {

        const texte1 = trans1.transcription_lignes
            .map(l => l.contenu).join(' ');
        const texte2 = trans2.transcription_lignes
            .map(l => l.contenu).join(' ');

        // Similarité simple (Levenshtein ou Jaccard)
        const similarite = this.calculerSimilarite(texte1, texte2);

        // Si les deux transcriptions sont très similaires (>90%)
        // → Probablement juste une meilleure photo (pas de triche)
        if (similarite > 0.90) {
            return {
                correction_detectee: false,
                raison:  'Photo reprise pour meilleure lisibilité',
            };
        }

        // Si les transcriptions diffèrent significativement (<90%)
        // → L'élève a probablement modifié sa réponse
        return {
            correction_detectee: true,
            similarite:         similarite,
            raison:             'Réponse modifiée entre les deux photos',
        };
    }

    static calculerSimilarite(str1, str2) {
        // Jaccard similarity sur les tokens
        const tokens1 = new Set(str1.split(/\s+/));
        const tokens2 = new Set(str2.split(/\s+/));
        const intersection = new Set(
            [...tokens1].filter(t => tokens2.has(t))
        );
        const union = new Set([...tokens1, ...tokens2]);
        return intersection.size / union.size;
    }
}
