## ADAPTATION PAR MATIÈRE — Le diable est dans les détails

### MATHÉMATIQUES

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  🔢 MATHÉMATIQUES — LA MATIÈRE DE LA LOGIQUE PURE              │
│                                                                 │
│  NATURE DE LA MATIÈRE :                                         │
│  → 90% raisonnement + 10% mémorisation                        │
│  → Très peu de vocabulaire à retenir                           │
│  → Beaucoup de MÉTHODES à maîtriser                           │
│  → Les erreurs sont binaires (juste ou faux)                  │
│  → Chaque chapitre dépend des précédents (séquentiel)         │
│                                                                 │
│  CE QUI MARCHE LE MIEUX :                                      │
│                                                                 │
│  1. CHAT SOCRATIQUE ✅✅✅ (méthode #1 pour les maths)         │
│     → L'élève doit DÉCOUVRIR les méthodes                     │
│     → L'IA guide par questions, pas par réponses              │
│     → Exemple :                                                │
│       "Tu veux montrer que f est croissante.                   │
│        Quel outil utilise-t-on pour étudier les variations ?" │
│       → L'élève : "La dérivée ?"                              │
│       → "Oui ! Calcule f'(x). Que doit-elle être pour que    │
│         f soit croissante ?"                                   │
│                                                                 │
│  2. CUED RECALL ✅✅✅ (méthode #2)                             │
│     → "La dérivée de ln(x) = ?"                               │
│     → "Si f est continue et f(a)×f(b) < 0, alors..."         │
│     → Réponses courtes, vérifiables, essentielles             │
│     → Flashcards des formules clés (SM-2)                     │
│                                                                 │
│  3. EXERCICES ANNALES ✅✅✅ (méthode #3)                       │
│     → Les maths au BAC = exercices types reconnaissables      │
│     → L'élève qui a fait 50 annales reconnaît les patterns    │
│     → L'IA corrige ÉTAPE PAR ÉTAPE, pas juste le résultat    │
│                                                                 │
│  CE QUI MARCHE MOINS BIEN :                                    │
│                                                                 │
│  → QCM seul (insuffisant pour les maths, les raisonnements    │
│    ne rentrent pas dans un QCM)                               │
│  → Free Recall seul ("explique les suites" = trop vague)     │
│  → SM-2 seul (les maths = pratique, pas mémorisation)        │
│                                                                 │
│  ADAPTATION DE LA MÉTHODE :                                    │
│  → 70% Drill (exercices résolus étape par étape)              │
│  → 20% Chat Socratique (quand l'élève bloque)                │
│  → 10% Cued Recall (formules essentielles)                    │
│                                                                 │
│  PROMPT IA ADAPTÉ MATHS :                                      │
│                                                                 │
│  "Tu es un tuteur de maths du BAC algérien.                   │
│   En maths, la priorité est le RAISONNEMENT.                  │
│   Quand tu évalues une réponse :                               │
│   → Accorde 70% de la note à la MÉTHODE                      │
│   → Accorde 30% à la RÉPONSE FINALE                          │
│   Un bon raisonnement avec erreur de calcul = 6/10            │
│   Un résultat correct sans méthode = 3/10"                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### PHYSIQUE-CHIMIE

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ⚡ PHYSIQUE-CHIMIE — LA MATIÈRE DU PONT THÉORIE↔RÉEL          │
│                                                                 │
│  NATURE DE LA MATIÈRE :                                         │
│  → 50% raisonnement + 30% mémorisation + 20% application     │
│  → Beaucoup de FORMULES à connaître                           │
│  → Les formules ont un SENS PHYSIQUE (pas juste des lettres)  │
│  → Les exercices = problèmes concrets (circuits, mouvements)  │
│  → Mélange de calcul ET de compréhension                      │
│                                                                 │
│  CE QUI MARCHE LE MIEUX :                                      │
│                                                                 │
│  1. TECHNIQUE FEYNMAN ✅✅✅ (méthode #1 pour la physique)     │
│     → La physique a du SENS concret                           │
│     → Si l'élève peut expliquer un phénomène                  │
│       sans formules → il a vraiment compris                   │
│     → Exemple :                                                │
│       KHAWARIZMI : "Explique-moi la chute libre               │
│                     comme à un enfant de 10 ans."             │
│       ÉLÈVE : "C'est quand tu lâches un objet.               │
│                Il tombe de plus en plus vite                   │
│                parce que la terre le tire vers elle.           │
│                Plus il tombe longtemps, plus il va vite."     │
│       IA : Score 8/10 ✅ "Tu as compris l'accélération !"     │
│                                                                 │
│  2. CHAT SOCRATIQUE ✅✅✅ (méthode #2)                        │
│     → Guide l'élève vers la bonne formule                     │
│     → Exemple :                                                │
│       "Un résistor de 100Ω avec I = 0.5A. Trouve U."         │
│       IA : "Quelle grandeur relie R, I et U ?"              │
│       Élève : "La loi d'Ohm ?"                               │
│       IA : "Oui ! Écris-la. Remplace les valeurs."           │
│                                                                 │
│  3. CUED RECALL ✅✅✅ (méthode #3 — formules)                 │
│     → "La relation entre période et fréquence : T = ?"       │
│     → "L'énergie cinétique : Ec = ?"                          │
│     → "La loi de Coulomb : F = ?"                             │
│     → Les formules DOIVENT être mémorisées (SM-2)            │
│                                                                 │
│  ADAPTATION DE LA MÉTHODE :                                    │
│  → 50% Drill (exercices avec formules appliquées)             │
│  → 25% Feynman (expliquer les phénomènes physiques)          │
│  → 15% Chat Socratique (quand l'élève bloque)                │
│  → 10% Cued Recall (formules à mémoriser par SM-2)           │
│                                                                 │
│  PROMPT IA ADAPTÉ PHYSIQUE :                                   │
│                                                                 │
│  "Tu es un tuteur de physique du BAC algérien.                │
│   En physique, la priorité est le SENS PHYSIQUE.              │
│   Quand tu évalues :                                           │
│   → L'élève a-t-il choisi la bonne loi/formule ?             │
│   → A-t-il justifié POURQUOI cette formule ?                 │
│   → A-t-il vérifié les UNITÉS ?                              │
│   → Le résultat est-il physiquement cohérent ?               │
│   Un résultat de 500 km/s pour une voiture = 0/10            │
│   même si le calcul est mathématiquement correct."            │
│                                                                 │
│  SPÉCIFICITÉ PHYSIQUE — LA VÉRIFICATION DES UNITÉS :          │
│                                                                 │
│  L'IA vérifie automatiquement :                               │
│  → L'élève a-t-il mis les unités dans sa réponse ?           │
│  → Les unités sont-elles cohérentes ?                         │
│  → L'ordre de grandeur est-il réaliste ?                     │
│                                                                 │
│  Exemple :                                                      │
│  Réponse élève : "U = 50"                                     │
│  IA : "50 quoi ? Volts ? Watts ? Les unités sont             │
│        OBLIGATOIRES en physique. U = 50 V ✅"                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## IMPLÉMENTATION TECHNIQUE — Comment l'IA s'adapte par matière

```javascript
// subject-config.ts — Configuration par matière

const SUBJECT_CONFIG = {

    mathematiques: {
        exerciseDistribution: {
            drill_step_by_step:  0.70,    // 70% exercices résolus étape par étape
            chat_socratique:     0.20,    // 20% quand l'élève bloque
            cued_recall:         0.10,    // 10% formules essentielles
        },
        evaluationWeights: {
            raisonnement:        0.70,    // 70% de la note = méthode
            resultat_final:      0.30,    // 30% = résultat correct
        },
        sm2_priority:         'low',      // SM-2 moins important (pratique > mémoire)
        feynman_frequency:    'weekly',   // 1x/semaine Feynman
        unit_check:           false,      // Pas de vérification unités
        step_by_step_eval:    true,       // Évaluation CHAQUE étape du raisonnement
    },

    physique: {
        exerciseDistribution: {
            drill_formulas:      0.50,    // 50% exercices avec formules
            feynman:             0.25,    // 25% expliquer les phénomènes
            chat_socratique:     0.15,    // 15% guidage
            cued_recall:         0.10,    // 10% formules clés (SM-2)
        },
        evaluationWeights: {

```javascript
// error-autopsy.js — Classification automatique des erreurs

class ErrorAutopsy {

    static async diagnose(userId, exerciseResult, openai, db) {

        if (exerciseResult.score >= 0.6) return null; // Pas d'erreur

        // Poser les 3 questions à l'élève
        const diagnosis = await this.askDiagnosticQuestions(
            userId,
            exerciseResult
        );

        // Classifier l'erreur
        const errorType = this.classify(diagnosis);

        // Prescrire le traitement
        const prescription = this.prescribe(errorType, exerciseResult);

        // Sauvegarder dans le profil d'erreurs de l'élève
        await db.query(`
            INSERT INTO error_profile
                (user_id, matiere, chapitre, error_type,
                 question_id, created_at)
            VALUES ($1, $2, $3, $4, $5, NOW())
        `, [
            userId,
            exerciseResult.matiere,
            exerciseResult.chapitre,
            errorType,
            exerciseResult.questionId,
        ]);

        // Ajuster automatiquement le prochain exercice
        await this.adjustNextSession(userId, errorType, db);

        return { errorType, prescription };
    }

    static classify(diagnosis) {
        if (!diagnosis.knewWhat)       return 'CONCEPT';
        if (!diagnosis.knewHow)        return 'METHOD';
        if (diagnosis.calculationError) return 'EXECUTION';
        return 'PANIC';
    }

    static prescribe(errorType, result) {

        const prescriptions = {
            CONCEPT: {
                nextAction:   'feynman_session',
                message:      `Tu n'as pas encore compris le concept de 
                               "${result.chapitre}". 
                               Avant tout exercice : explique-le moi 
                               avec tes mots.`,
                sm2_reset:    true,   // Revenir à intervalle = 1 jour
                urgency:      'HIGH',
            },
            METHOD: {
                nextAction:   'method_drill',
                message:      `Tu connais le concept mais tu ne sais pas 
                               quand l'utiliser. 
                               Prochain drill : 10 exercices 
                               "Quelle méthode ?" (sans calcul).`,
                sm2_reset:    false,
                urgency:      'MEDIUM',
            },
            EXECUTION: {
                nextAction:   'calculation_drill',
                message:      `Tu sais quoi faire et comment, 
                               mais tu fais des erreurs de calcul.
                               Focus : vérification systématique 
                               après chaque étape.`,
                sm2_reset:    false,
                urgency:      'LOW',
            },
            PANIC: {
                nextAction:   'exam_simulation',
                message:      `Tu sais faire mais tu bloques sous pression.
                               Prochain entraînement : Mode Examen 
                               avec chronomètre. 45 min. Aucune aide.`,
                sm2_reset:    false,
                urgency:      'MEDIUM',
            },
        };

        return prescriptions[errorType];
    }

    // Tracker les patterns d'erreur sur 30 jours
    static async getErrorProfile(userId, matiere, db) {

        const errors = await db.query(`
            SELECT error_type, COUNT(*) as count,
                   chapitre
            FROM error_profile
            WHERE user_id = $1
              AND matiere  = $2
              AND created_at > NOW() - INTERVAL '30 days'
            GROUP BY error_type, chapitre
            ORDER BY count DESC
        `, [userId, matiere]);

        // Si 60%+ des erreurs = TYPE 1 (CONCEPT)
        // → L'élève a un problème fondamental de compréhension
        // → Recommandation : revoir les bases avant les exercices

        const dominantError = errors.rows[0]?.error_type;
        const weakestChapter = errors.rows[0]?.chapitre;

        return {
            dominantError,
            weakestChapter,
            rawData: errors.rows,
        };
    }
}

│  → Le résultat est-il proche de mon estimation ?              │
│  → Physiquement, ce résultat a-t-il du sens ?                 │
│  → Si non → reprendre l'étape 4                                │
│                                                                 │
│  POURQUOI C'EST NOUVEAU :                                       │
│  → Aucune app ne force l'étape ESTIMATION avant le calcul     │
│  → L'estimation développe l'intuition physique                 │

```javascript
// sens-avant-calcul.js — Protocole physique en 5 étapes

class SensAvantCalcul {

    static EXERCISE_TEMPLATE = {

        step1_schema: {
            instruction: `
                Avant de calculer quoi que ce soit :
                📐 DESSINE la situation physique.
                Identifie toutes les grandeurs sur ton schéma.
                Décris ton schéma en 1-2 phrases.
            `,
            timeLimit:   60,    // secondes
            required:    true,
        },

        step2_sens: {
            instruction: `
                En une phrase simple :
                🌍 Que se passe-t-il PHYSIQUEMENT dans cet exercice ?
                (Comme si tu l'expliquais à quelqu'un sans formules)
            `,
            timeLimit:   45,
            required:    true,
            evaluation: `
                L'élève a-t-il décrit le phénomène physique correctement ?
                Pas les formules. Le SENS.
                Score /3 : 
                3 = description physique parfaite
                2 = description partielle
                1 = description confuse
                0 = formule récitée (sans compréhension du phénomène)
            `,
        },

        step3_estimation: {
            instruction: `
                🎯 Avant de calculer, ESTIME l'ordre de grandeur :
                Le résultat sera probablement :
                □ Quelques unités (1-10)
                □ Dizaines (10-100)
                □ Centaines (100-1000)
                □ Millièmes (0.001-0.01)
                
                Et l'unité probable : ________
                
                Note : Il n'y a pas de "mauvaise" estimation.
                L'objectif est de RÉFLÉCHIR avant de calculer.
            `,
            timeLimit:   45,
            required:    true,
        },

        step4_calcul: {
            instruction: `🔢 Maintenant calcule. Montre toutes les étapes.`,
            timeLimit:   300,
            required:    true,
        },

        step5_verification: {
            instruction: `
                ✅ VÉRIFICATION FINALE :
                1. Ton résultat est-il proche de ton estimation ? (Oui/Non)
                2. Le résultat est-il physiquement cohérent ?
                   (Ex: une vitesse de 5,000 km/s pour une voiture = impossible)
                3. Les unités sont-elles correctes ?
            `,
            timeLimit:   30,
            required:    true,
        },
    };

    // Évaluation de l'estimation (après avoir le vrai résultat)
    static evaluateEstimation(
        studentEstimation,
        correctAnswer
    ) {
        const ratio = correctAnswer / studentEstimation;

        if (ratio >= 0.1 && ratio <= 10) {
            return {
                score:    3,
                feedback: "✅ Excellente intuition physique !",
            };
        } else if (ratio >= 0.01 && ratio <= 100) {
            return {
                score:    2,
                feedback: "🔶 Estimation dans la bonne zone.",
            };
        } else {
            return {
                score:    1,
                feedback: `🔴 Ton estimation était très loin.
                           C'est normal au début.
                           L'intuition physique se développe avec la pratique.
                           Note mentalement cet écart.`,
            };
        }
    }
}

// exercise-generator.js — Génération d'exercices originaux

class ExerciseGenerator {

    static async generateBatch(
        chapitre,     // ex: PROGRAMME_MATHS_3AS.sections[0].chapitres[0]
        annaleStyle,  // ex: structure d'un exercice d'annale similaire
        count,        // ex: 20 exercices
        difficulte,   // 'facile' | 'moyen' | 'difficile' | 'bac'
        openai
    ) {
        const prompt = `
Tu es un créateur d'exercices pour le BAC algérien.

CHAPITRE : ${chapitre.titre}
MATIÈRE : ${chapitre.matiere || 'Mathématiques'}
OBJECTIFS D'APPRENTISSAGE :
${chapitre.objectifs.map((o, i) => `${i+1}. ${o}`).join('\n')}

FORMULES DISPONIBLES :
${chapitre.formules.map(f => `${f.nom} : ${f.formule}`).join('\n')}

ERREURS FRÉQUENTES DES ÉLÈVES :
${chapitre.erreurs_frequentes.join('\n')}

DIFFICULTÉ DEMANDÉE : ${difficulte}

STYLE : Exercice typique du BAC algérien.
Ressemble aux vrais sujets dans la formulation.

GÉNÈRE ${count} exercices ORIGINAUX en JSON :
[
    {
        "question": "<énoncé complet>",
        "sous_questions": [
            {
                "question": "<sous-question>",
                "reponse": "<réponse détaillée étape par étape>",
                "methode": "<quelle méthode utiliser et pourquoi>",
                "piege": "<erreur fréquente à éviter ici>",
                "points": <1-4>
            }
        ],
        "chapitre_id": "${chapitre.id}",
        "difficulte": "${difficulte}",
        "duree_estimee_minutes": <5-20>,
        "objectifs_testes": ["<objectif 1>", "<objectif 2>"]
    }
]

RÈGLES :
→ Les exercices doivent être ORIGINAUX (pas copiés d'annales existantes)
→ Les valeurs numériques doivent être réalistes
→ Chaque exercice teste au moins 1 objectif du programme
→ Varier les types : calcul, démonstration, étude de cas
→ Le piège doit correspondre aux erreurs fréquentes RÉELLES
`;

        const response = await openai.chat.completions.create({
            model:           'gpt-4o-mini',
            messages:        [{ role: 'user', content: prompt }],
            temperature:     0.7,     // Varié pour diversité
            response_format: { type: 'json_object' },
        });

        return JSON.parse(response.choices[0].message.content);
    }
}

// exercise-generator.js — Génération d'exercices originaux

class ExerciseGenerator {

    static async generateBatch(
        chapitre,     // ex: PROGRAMME_MATHS_3AS.sections[0].chapitres[0]
        annaleStyle,  // ex: structure d'un exercice d'annale similaire
        count,        // ex: 20 exercices
        difficulte,   // 'facile' | 'moyen' | 'difficile' | 'bac'
        openai
    ) {
        const prompt = `
Tu es un créateur d'exercices pour le BAC algérien.

CHAPITRE : ${chapitre.titre}
MATIÈRE : ${chapitre.matiere || 'Mathématiques'}
OBJECTIFS D'APPRENTISSAGE :
${chapitre.objectifs.map((o, i) => `${i+1}. ${o}`).join('\n')}

FORMULES DISPONIBLES :
${chapitre.formules.map(f => `${f.nom} : ${f.formule}`).join('\n')}

ERREURS FRÉQUENTES DES ÉLÈVES :
${chapitre.erreurs_frequentes.join('\n')}

DIFFICULTÉ DEMANDÉE : ${difficulte}

STYLE : Exercice typique du BAC algérien.
Ressemble aux vrais sujets dans la formulation.

GÉNÈRE ${count} exercices ORIGINAUX en JSON :
[
    {
        "question": "<énoncé complet>",
        "sous_questions": [
            {
                "question": "<sous-question>",
                "reponse": "<réponse détaillée étape par étape>",
                "methode": "<quelle méthode utiliser et pourquoi>",
                "piege": "<erreur fréquente à éviter ici>",
                "points": <1-4>
            }
        ],
        "chapitre_id": "${chapitre.id}",
        "difficulte": "${difficulte}",
        "duree_estimee_minutes": <5-20>,
        "objectifs_testes": ["<objectif 1>", "<objectif 2>"]
    }
]

RÈGLES :
→ Les exercices doivent être ORIGINAUX (pas copiés d'annales existantes)
→ Les valeurs numériques doivent être réalistes
→ Chaque exercice teste au moins 1 objectif du programme
→ Varier les types : calcul, démonstration, étude de cas
→ Le piège doit correspondre aux erreurs fréquentes RÉELLES
`;

        const response = await openai.chat.completions.create({
            model:           'gpt-4o-mini',
            messages:        [{ role: 'user', content: prompt }],
            temperature:     0.7,     // Varié pour diversité
            response_format: { type: 'json_object' },
        });

        return JSON.parse(response.choices[0].message.content);
    }
}
