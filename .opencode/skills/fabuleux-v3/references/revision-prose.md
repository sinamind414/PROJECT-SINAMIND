# Révision prose : la passe de soustraction

> Route PROSE de `fabuleux`. Le mode d'échec à tuer : un modèle « qui se force à bien faire »
> produit **plus long, plus structuré, plus de tableaux**, et un lecteur avec du goût trouve ça
> gonflé, pas meilleur. La discipline n'ajoute pas : elle **enlève**.

## Le principe

Un premier jet est presque toujours trop long et trop habillé. La valeur se révèle en **coupant**,
pas en empilant. Vise : *plus court, plus net, plus naturel*, à sens égal.

```
CRITÈRES + cible de longueur → DRAFT → SOUSTRAIRE (~20%) → NETTOYER (anti-slop) → VÉRIFIER
```

## La passe de soustraction (concrète)

Après le draft, relire une fois avec un seul objectif : **retirer**.

- **Couper ~20 % des mots** sans perdre de sens. Si tu n'y arrives pas, c'est que le draft était déjà nu (rare).
- **Tuer les fillers** : « en fait », « il est important de noter que », « dans le cadre de », « afin de », « permettre de », « à terme », « globalement ».
- **Défaire la sur-structure** : pas de titre `##`, de tableau, de liste numérotée **si le texte se tient en paragraphes**. Un email n'est pas un rapport. Un tweet n'est pas un plan.
- **Défaire le staccato** : relier les phrases hachées avec *mais / et / c'est* quand le rythme le demande (sauf effet voulu).
- **Une idée = une formulation** : si une idée est dite deux fois, garder la meilleure.
- **Verbes pleins, voix active** : « nous décidons » plutôt que « il a été décidé » ; « teste » plutôt que « procéder à un test ».
- **Match du registre** : le texte parle comme la personne parlerait, pas comme un communiqué.

## La passe anti-slop (autonome)

Après avoir coupé, nettoie les tics de langage des modèles :
- **Pas de négation-contraste** (« ce n'est pas X, c'est Y ») : préférer deux clauses positives parallèles.
- **Mène par la réponse** : la conclusion d'abord, le contexte ensuite.
- **Bannir les ouvertures vides** : « Excellente question », « Il est important de noter que », « En conclusion ».
- **Pas de récapitulatif inutile** ni de péroraison qui répète ce qui vient d'être dit.
- **Jargon → mot simple** ; anglicisme paresseux → équivalent courant.
- **La profondeur épouse la demande** : une question simple appelle une réponse courte.

## Ponctuation FR
- **Zéro cadratin `—`** : remplacer par `,` `:` `.` ou des parenthèses.
- Espace insécable avant `: ; ? !` sur une surface soignée.

## Vérifier (les critères, pas l'impression)
- La demande est-elle **entièrement** traitée ? (pas juste joliment tournée)
- Zéro affirmation fausse ou inventée ?
- Compté : la sortie est-elle **plus courte** que le draft ? Plus naturelle à lire à voix haute ?
- Test honnête : si je montrais les deux versions à un lecteur pressé, laquelle préfère-t-il ? Si c'est le draft, j'ai trop coupé ; si c'est la version courte (le cas normal), je livre.

> Rappel : sur la prose one-shot, un bon modèle est déjà près du plafond. L'objectif de la route prose
> n'est pas de « battre » le défaut par plus de matière, c'est de **ne pas le gonfler** et de garantir
> l'honnêteté + la concision. Le vrai gain de `fabuleux` est ailleurs (artefact, vérification, vision).
